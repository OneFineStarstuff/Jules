# Global AGI and AI Regulatory Harmonization Matrix (2026–2030)

## 1. Primary Prudential and Conduct Mandates

For Global Systemically Important Financial Institutions (G-SIFIs), AI governance is an extension of established risk management regimes.

| Jurisdiction | Regulation / Framework | Core Technical Requirement for AI |
| :--- | :--- | :--- |
| **Global** | **Basel III / BCBS 239** | Data lineage and operational resilience for autonomous capital allocation. |
| **United States** | **SR 11-7 (Fed/OCC)** | Model Risk Management (MRM); requirement for continuous back-testing and conceptual soundness verification. |
| **European Union** | **EU AI Act** | High-Risk System classification; mandatory transparency logs and "Human-on-the-loop" (HOTL) oversight. |
| **United Kingdom** | **SMCR / PRA / FCA** | Individual accountability for "Algorithmic Outcomes"; Senior Manager sign-off on AI decision-engines. |
| **Singapore** | **MAS FEAT / Veritas** | Fairness, Ethics, Accountability, and Transparency (FEAT) metrics; bias mitigation in credit scoring. |
| **Hong Kong** | **HKMA Expectations** | Rigorous security auditing for "Autonomous Customer Interaction" platforms. |

## 2. Technical Standards and Security Frameworks

- **ISO/IEC 42001 (AIMS)**: The foundational management system for AI risk.
- **NIST AI RMF 1.0**: Providing the "Govern, Map, Measure, Manage" lifecycle for trust-by-design.
- **ISO 31000**: Enterprise-wide risk management integration.
- **US EO 14110**: National standards for "Red-Teaming" dual-use foundation models.

## 3. The Institutional Mandate
G-SIFIs must reconcile **Recursive Autonomy** (Self-improving agents) with **Tiered Administration**. This requires the implementation of **"Governance-as-Code" (GaC)** where regulatory constraints are compiled into deterministic guardrails (GDL/OPA) that bound probabilistic LLM reasoning.
